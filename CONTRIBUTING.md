# Contribuindo

Obrigado por considerar contribuir para o projeto!

Guia rápido:

- Abra uma issue clara descrevendo o problema ou a melhoria proposta.
- Faça um fork do repositório e crie uma branch com nome significativo: `feature/<descrição>` ou `bugfix/<descrição>`.
- Mantenha commits pequenos e com mensagens descritivas.
- Envie um Pull Request com descrição completa: o que foi alterado, por que e como testar.

Pontos a observar:

- Teste alterações em um ambiente de desenvolvimento antes de submeter.
- Siga os padrões de nomenclatura e estilo do projeto.
- Mantenedores revisarão PRs e poderão solicitar mudanças.

Agradecemos sua contribuição!
